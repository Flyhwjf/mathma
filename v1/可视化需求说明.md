# 妈妈杯A题：智慧物流车辆路径规划与调度可视化需求说明

## 概述
本文档说明四个问题中需要绘制的图表及其目的。所有图表已在`start.ipynb`中实现，使用Matplotlib绘制。

## 问题1：不考虑时间窗和容量约束的单车辆调度（基础TSP模型）

### 1. 哈密顿量演化曲线
- **图表类型**：折线图
- **X轴**：迭代次数
- **Y轴**：哈密顿量（能量）
- **目的**：展示模拟退火算法优化过程中能量（目标函数值）的下降过程，验证算法收敛性
- **关键观察点**：
  - 初始能量值
  - 收敛趋势（能量下降速度）
  - 最终稳定能量值
- **实现函数**：`simulate_annealing_process()`

### 2. 路径可视化图
- **图表类型**：散点图+路径连线图
- **内容**：
  - 客户点位置（基于MDS从距离矩阵生成的2D坐标）
  - 仓库位置（原点）
  - 车辆行驶路径（带箭头指示方向）
- **目的**：直观展示最优访问顺序和路径走向
- **关键观察点**：
  - 路径是否合理（避免交叉、过长跳跃）
  - 客户点分布
  - 起点和终点均为仓库
- **实现函数**：`visualize_route()`

### 3. 输出信息（非图表）
- 车辆行驶路线序列
- 总运输时间
- 约束验证结果（是否所有客户点都被访问且只访问一次）

## 问题2：考虑时间窗惩罚、无容量约束的单车辆调度（带软时间窗的TSP）

### 1. 哈密顿量演化曲线（带时间窗惩罚）
- **图表类型**：折线图
- **颜色**：绿色（区别于问题1）
- **目的**：展示加入时间窗惩罚后的优化过程，对比与问题1的能量尺度差异
- **关键观察点**：
  - 能量值是否更高（因加入惩罚项）
  - 收敛是否平稳

### 2. 路径可视化图（带时间窗信息）
- **图表类型**：散点图+路径连线图
- **新增元素**：在标题中显示总时间窗惩罚
- **目的**：直观展示考虑时间窗后的最优路径

### 3. 开始时间与时间窗对比图
- **图表类型**：双线填充图
- **子图1**：
  - 填充区域：各客户点的时间窗范围（绿色半透明）
  - 折线：实际开始服务时间（蓝色点线）
- **目的**：可视化时间窗违反情况
- **关键观察点**：
  - 开始时间是否在时间窗内
  - 早到或晚到的程度

### 4. 时间窗惩罚分布图
- **图表类型**：柱状图
- **X轴**：访问顺序
- **Y轴**：时间窗惩罚值
- **颜色**：红色（有惩罚），绿色（无惩罚）
- **目的**：显示各客户点的惩罚贡献
- **关键观察点**：
  - 哪些客户点导致主要惩罚
  - 惩罚的分布模式

### 5. 输出信息（非图表）
- 车辆行驶路线
- 每个客户点的开始服务时间及违反程度（早到/晚到）
- 总运输时间
- 总时间窗惩罚
- 综合目标值（运输时间+惩罚）

## 通用可视化设置

### 1. 坐标生成方法
- 使用多维缩放（MDS）从距离矩阵生成2D坐标
- 仓库固定为原点(0,0)
- **注意**：生成坐标仅为可视化目的，与实际地理坐标可能有差异

### 2. 图表样式统一设置
- 中文字体支持（SimHei）
- 网格线（alpha=0.3）
- 适当的图形尺寸（10×6英寸或12×8英寸）
- 清晰图例
- 标题包含关键指标（总时间、总惩罚等）

### 3. 颜色方案
- 仓库：黑色方块
- 客户点：按簇或车辆分配颜色
- 路径：与所属车辆/簇同色
- 时间窗区域：绿色半透明填充

### 4. 交互性说明
- 所有图表为静态PNG格式
- Jupyter Notebook中可直接查看
- 可调整参数重新生成

## 注意事项

1. **坐标真实性**：MDS生成的坐标仅为可视化近似，不反映真实地理位置
2. **计算时间**：大规模问题（50客户点）可能需要较长时间求解
3. **参数调优**：惩罚系数、聚类数量等参数需要根据实际结果调整
4. **随机性**：模拟退火算法有随机性，每次运行结果可能略有不同
5. **硬件限制**：实际量子计算部分（如使用Kaiwu SDK真机）需考虑550量子比特限制

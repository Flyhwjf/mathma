## 概述

本文档设计问题3的大规模单车辆调度算法解决方案。针对50个客户点（N=50）导致QUBO变量数达2500的超规模问题，采用**迭代聚类窗口法**的混合量子-经典策略，将变量数控制在量子求解器可接受范围内。

**Why:** 直接QUBO建模需要2500个变量，远超550量子比特真机限制。必须采用降维分解策略。

**How to apply:** 通过时空联合聚类将问题分解为3-4个子问题，在每个子问题中使用滑动窗口局部优化，通过迭代重新聚类减少层间误差。

## 1. 总体架构

### 1.1 算法流程

```
数据加载 → 时空联合聚类 → 簇间路径规划 → 簇内滑动窗口优化 → 评估与重新聚类 → 迭代改进 → 路径拼接
```

**设计决策:** 采用"聚类+滑动窗口"的混合方法，而非单一策略。

**Why:** 纯聚类方法层间误差大，纯滑动窗口方法全局性差。混合方法平衡了全局结构和局部优化。

**How to apply:** 先聚类获得全局结构，再用滑动窗口精细优化每个簇，通过迭代减少边界误差。

### 1.2 变量规模控制

| 阶段 | 最大同时变量数 | 说明 |
|------|----------------|------|
| 直接建模 | 2500 | N=50, N²=2500，超限 |
| 簇内完整优化 | 225 | 每簇约15点，15²=225，符合550限制 |
| 滑动窗口优化 | 100 | 窗口大小10，10²=100，显著低于550限制 |

**Why:** 每个QUBO子问题的变量数必须≤550。簇内优化225变量，窗口优化100变量，均满足要求。

**How to apply:** 优先使用窗口优化（100变量），对复杂簇使用完整簇优化（225变量）。

**How to apply:** 确保每个QUBO子问题变量数不超过550，优先保证量子求解可行性。

## 2. 时空联合聚类算法

### 2.1 距离度量

```
D(i, j) = 0.6 × Norm_dist(d_ij) + 0.4 × Norm_time(|a_i - a_j| + |b_i - b_j|)
```

- `d_ij`: 节点i到j的旅行时间
- `a_i, b_i`: 时间窗下界和上界
- `Norm_dist(x)`: 距离归一化，`(x - min_d) / (max_d - min_d)`，其中`min_d`/`max_d`为所有`d_ij`的最小/最大值
- `Norm_time(x)`: 时间归一化，`(x - min_t) / (max_t - min_t)`，其中`min_t`/`max_t`为所有时间窗差异的最小/最大值

**设计决策:** 时空权重比6:4。

**Why:** 地理位置主导但时间窗约束重要。实验表明6:4平衡空间邻近和时间窗一致性。

**How to apply:** 在聚类前计算所有节点对的时空距离矩阵。

### 2.2 聚类参数

- 簇数K: 3-4个（基于50个点规模）
- 算法: 改进K-means
- 初始中心: 基于时空距离最分散的点
- 停止条件: 簇分配变化<5%或最大迭代50次

**Why:** 3-4个簇使每簇12-17个点，适合后续QUBO求解。

**How to apply:** 实现自定义K-means，使用时空距离度量替代欧氏距离。

### 2.3 输出要求

- 每个节点的簇标签
- 簇中心坐标（时空平均）
- 簇内点的时间窗重叠度

## 3. 滑动窗口优化机制

### 3.1 窗口参数

- 窗口大小: 10个客户点
- 滑动步长: 5个客户点（50%重叠）
- 锚定点: 窗口两侧各1个固定点
- 优化方法: 问题2的混合量子-经典算法

**设计决策:** 50%重叠窗口。

**Why:** 充分重叠确保平滑过渡，减少边界效应。

**How to apply:** 对每个簇，从左到右滑动窗口，前窗口结果作为后窗口输入。

### 3.2 窗口内QUBO建模

复用问题2的混合算法：

1. **经典侧**: 预计算修正距离矩阵（λ=1.0）
2. **量子侧**: 构建TSP QUBO模型（100个变量）
3. **求解**: Kaiwu SDK模拟退火
4. **后处理**: 计算实际开始时间和惩罚

**Why:** 重用已验证代码，确保时间窗惩罚正确处理。

**How to apply:** 将问题2的核心函数封装为可重用的优化模块。

### 3.3 窗口滑动策略

```
for cluster in clusters:
    path = initialize_with_nearest_neighbor(cluster)
    cluster_size = len(cluster)
    window_size = min(10, cluster_size)  # 适应小簇
    step_size = max(1, window_size // 2)  # 50%重叠
    
    for window_start in range(0, cluster_size - window_size + 1, step_size):
        window_points = path[window_start:window_start + window_size]
        optimized_window = solve_window_QUBO(window_points)
        path[window_start:window_start + window_size] = optimized_window
```

## 4. 迭代改进过程

### 4.1 重新聚类触发条件

1. **边界违反严重**: 簇边界点的时间窗惩罚 > 平均惩罚的2倍
2. **路径交叉**: 相邻簇间路径交叉长度 > 簇间距离的20%
3. **簇不平衡**: 某个簇的综合代价 > 平均代价的1.5倍

**设计决策:** 基于优化结果动态调整簇边界。

**Why:** 初始聚类可能不是最优划分，需要根据优化结果调整。

**How to apply:** 监控边界点性能，触发重新分配。

### 4.2 簇边界调整策略

1. **边界点迁移**: 将高惩罚边界点重新分配到相邻簇
2. **簇合并**: 如果两个簇间路径交叉严重，合并为一簇
3. **簇分裂**: 如果簇内路径过长，在中间点处分裂

**设计决策:** 渐进式调整而非完全重新聚类。

**Why:** 保持大部分优化结果，只调整问题区域。

**How to apply:** 实现边界点评估函数和迁移算法。

### 4.3 收敛判断

- 最大迭代次数: 5
- 综合代价变化阈值: <1%（综合代价 = 总运输距离 + 总时间窗惩罚）
- 簇分配稳定度: >95%节点簇标签不变

**Why:** 平衡优化质量和计算时间。

**How to apply:** 记录每次迭代的目标函数和簇分配，检测收敛。
